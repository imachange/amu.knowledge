# 0001. ADR (Any Decision Record) の導入

日付: 2026-02-19

## ステータス

承認済み

## 背景

AMU プロジェクトでは、アーキテクチャ・プロセス・ツール選定・運用方針など様々な決定を行う。
しかし、「なぜそう決めたか」を記録しないと、後から理由が分からなくなる問題があった。

決定の背景と理由を残す仕組みとして、ADR (Any Decision Record) の導入を検討した。

## 検討した選択肢

- ADR を導入する
- 導入しない（都度 Issue や Discussion で議論・記録する）

## 決定

ADR を導入し、`_amu/docs/adr/` に Markdown ファイルとして蓄積していく。

- ファイル命名規則: `0001-kebab-case-title.md`（番号は4桁ゼロパディング）
- テンプレート: `_amu/docs/adr/template.md` を使用する

## 理由

ADR を導入する理由:
- 「なぜ、こうしたんだっけ？」が可視化できる
- コードと一緒に Git で管理でき、PR でレビュー可能
- この手法を学習できる

導入しないという選択肢を選ばなかった理由:
- Issue や Discussion だけでは決定の経緯が散在し、後から追いにくい
